# Навигация и пользовательские сценарии

В приложении всего два маршрута, и переходы между ними простые. Навигационного
меню в привычном смысле нет — движение между экранами происходит через кнопки
действий и ссылки.

## Маршруты

| URL | Страница | Файл |
|---|---|---|
| `/` | Главная (лендинг) | `app/page.tsx` |
| `/room/[roomId]` | Комната видеозвонка | `app/room/[roomId]/page.tsx` |
| `/room/[roomId]?create=true` | Та же комната, но в режиме создания | тот же файл |

`roomId` — это код комнаты вида `XXXX-XXXX` (8 символов из набора без похожих
символов, с дефисом посередине, например `4X9T-8WJS`).

Параметр запроса `create=true` сообщает клиенту, что комнату нужно **создать**
(организатор), а не просто войти в существующую. Без него вход в несуществующую
комнату завершится ошибкой «Комната не найдена».

## Навигация в хедере главной страницы

В `site-header.tsx` есть навигационные ссылки «Возможности», «Тарифы»,
«Компания», а также переключатель языка «RU» (статичный бейдж) и кнопка «Войти».
Ссылки навигации ведут на `#` (заглушки-якоря) — отдельных страниц под них в
проекте нет. Кнопка «Войти» и бейдж «RU» не имеют обработчиков. Подробнее —
[`landing-page.md`](./landing-page.md).

## Сценарий 1. Начать звонок (организатор)

1. На главной пользователь нажимает кнопку **«Начать звонок»** в hero-секции.
2. Открывается диалог **«Новая конференция»** (`start-call-dialog.tsx`):
   - заранее сгенерирован код комнаты;
   - пользователь вводит своё имя;
   - код можно скопировать кнопкой «Скопировать».
3. По нажатию **«Начать конференцию»** имя сохраняется в `localStorage`, и
   браузер переходит на `/room/<код>?create=true`.
4. На странице комнаты клиент подключается к серверу и **создаёт** комнату.

## Сценарий 2. Войти по коду (участник)

1. На главной пользователь нажимает **«Войти по коду»**.
2. Открывается диалог **«Войти по коду»** (`join-call-dialog.tsx`):
   - поле имени;
   - поле кода с авто-форматированием в `XXXX-XXXX`.
3. По нажатию **«Войти в комнату»** (или Enter) имя сохраняется, и браузер
   переходит на `/room/<код>` (без `create=true`).
4. Клиент подключается к существующей комнате. Если её нет — экран ошибки.

Подробности обоих диалогов — в [`dialogs.md`](./dialogs.md).

## Сценарий 3. Внутри комнаты

- Подключение начинается автоматически при открытии страницы.
- Код комнаты в верхней панели можно скопировать одним кликом, чтобы
  пригласить других.
- Имя можно поменять кнопкой с карандашом — после сохранения страница
  перезагружается, чтобы другие участники увидели новое имя.
- Кнопка **«Покинуть комнату»** (красная трубка) разрывает соединение и
  возвращает пользователя на главную (`/`).

Полное описание экрана — в [`room-page.md`](./room-page.md).

## Состояния страницы комнаты

Страница `/room/[roomId]` показывает разный контент в зависимости от статуса
подключения:

- `idle` / `connecting` — спиннер и «Подключение к комнате…»;
- `connected` — собственно интерфейс звонка;
- `error` — текст ошибки и кнопка «На главную»;
- `disconnected` — «Вы покинули комнату» и кнопка «На главную».

Эти состояния приходят из хука `useMediasoup` — см.
[`client-logic.md`](./client-logic.md).
