# Дизайн-система

Описание визуального оформления сайта по тому, что задано в коде
(`app/globals.css`, `app/layout.tsx` и компоненты). Без добавления новых стилей.

## Тема

Сайт по умолчанию **тёмный**. Фон очень тёмный, текст почти белый. Цвета заданы
в пространстве OKLCH через CSS-переменные в `:root` (и отдельный набор для
`.dark`). Класс `bg-background` навешен на `<html>` в корневом layout.

### Основные токены (`:root`)

- `--background` — почти чёрный фон (`oklch(0.06 0 0)`);
- `--foreground` — почти белый текст (`oklch(0.985 0 0)`);
- `--card`, `--popover` — тёмные поверхности (`oklch(0.1 0 0)`);
- `--primary` — светлый (почти белый), `--primary-foreground` — тёмный
  (используется для логотипа и основной кнопки);
- `--secondary` / `--muted` / `--accent` — тёмно-серые поверхности
  (`oklch(0.16 0 0)`);
- `--muted-foreground` — приглушённый серый текст;
- `--destructive` — красный (для состояний mute/выключено и кнопки выхода);
- `--border`, `--input`, `--ring` — тонкие светлые границы с прозрачностью.

Палитра по сути монохромная (оттенки серого от чёрного к белому) плюс
**красный** акцент для деструктивных действий и **зелёный** для индикатора
говорящего/успеха (используется напрямую в компонентах, например
`text-green-500`, `ring-green-500`).

## Радиусы

Базовый радиус `--radius: 0.75rem`, на его основе вычисляются производные
размеры от `--radius-sm` до `--radius-4xl`. В интерфейсе активно используются
сильно скруглённые элементы: круглые кнопки управления в комнате, скруглённые
карточки преимуществ, «капсульные» бейджи и кнопки (`rounded-full`).

## Шрифты

Подключены два шрифта Google через `next/font` в `app/layout.tsx`:

- **Geist Sans** — основной шрифт (переменная `--font-geist-sans`,
  класс `font-sans`); также используется как заголовочный (`--font-heading`).
- **Geist Mono** — моноширинный (`--font-geist-mono`, класс `font-mono`),
  применяется для кода комнаты.

Связка задаётся в `@theme inline` в `globals.css`. Класс `font-sans`
применяется к `html` по умолчанию.

## Визуальные приёмы

- **Декоративные фоны**: мягкие радиальные свечения и едва заметные сетки в
  hero, блоке преимуществ и баннере качества — все помечены `aria-hidden`, т.е.
  чисто декоративные и не мешают скринридерам.
- **Бейджи-капсулы**: маленькие закруглённые метки с тонкой границей
  («Преимущества», «Качество видео», «RU», статус «В эфире» и т.п.).
- **Эффекты наведения**: лёгкий подъём карточек (`hover:-translate-y-1`),
  подсветка границ и «блик» в углу карточки.
- **Иконки**: только из `lucide-react`, единообразных размеров; эмодзи в роли
  иконок не используются.

## Доступность

- Семантические теги (`main`, `header`, `footer`, `nav`, `section`).
- Кнопки-иконки снабжены `aria-label` (микрофон, камера, демонстрация экрана,
  выход, копирование кода, регулятор громкости и т.д.).
- Декоративные элементы скрыты от скринридеров (`aria-hidden`).
- Сообщения об ошибках в форме входа имеют роль `alert`.
- Индикация состояния (mute/камера) дублируется иконками, а не только цветом.
