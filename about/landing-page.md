# Главная страница (лендинг)

Файл: `app/page.tsx`. Это маркетинговая страница, собранная из четырёх блоков,
которые рендерятся друг за другом внутри `<main>`:

```tsx
<SiteHeader />   // хедер
<Hero />         // первый экран с кнопками
<Features />     // блок преимуществ
<QualityBanner /> // баннер качества видео
```

Страница использует тёмную тему и фон `bg-background`. Ниже — назначение
каждого блока.

---

## 1. Хедер — `components/site-header.tsx`

Закреплён абсолютно поверх hero (`absolute inset-x-0 top-0`), занимает всю
ширину. Содержит:

- **Логотип** (`components/logo.tsx`) слева — иконка видеокамеры в квадрате +
  текст «Riplexo».
- **Навигация** (видна на десктопе, скрыта на мобильных): ссылки
  «Возможности», «Тарифы», «Компания» — все ведут на `#` (якоря-заглушки).
- **Справа**: статичный бейдж языка «RU» и кнопка-призрак **«Войти»**.

Назначение: брендинг и верхнеуровневая навигация. Кнопка «Войти» и языковой
бейдж визуально присутствуют, но не несут обработчиков.

---

## 2. Hero-секция — `components/hero.tsx`

Первый экран (`min-h-screen`), по центру по вертикали и горизонтали. Это
клиентский компонент (`"use client"`), потому что управляет открытием диалогов.

Элементы сверху вниз:

- **Декоративный фон**: мягкое радиальное свечение и едва заметная сетка
  (оба помечены `aria-hidden`, чисто визуальные).
- **Бейдж статуса**: пульсирующая точка + текст «В эфире — без установки
  приложений».
- **Заголовок H1**: «Бессрочные видеозвонки в один клик.»
- **Подзаголовок**: пояснение про отсутствие таймеров и вход по коду.
- **Две кнопки действия**:
  - **«Начать звонок»** (основная, с иконкой видео и стрелкой) — открывает
    диалог `StartCallDialog`.
  - **«Войти по коду»** (призрачная) — открывает диалог `JoinCallDialog`.

Логика переходов:

```ts
handleStart(code) → window.location.href = `/room/${code}?create=true`
handleJoin(code)  → window.location.href = `/room/${code}`
```

То есть сам hero — точка входа в оба пользовательских сценария. Диалоги
описаны в [`dialogs.md`](./dialogs.md).

---

## 3. Блок преимуществ — `components/features.tsx`

Секция с заголовком «Всё для общения. Ничего лишнего.» и подзаголовком. Над
заголовком — бейдж «Преимущества». Под ними — адаптивная сетка карточек
(1 колонка на мобильных, 2 на `sm`, 3 на `lg`).

Шесть карточек (каждая — иконка, заголовок, описание):

1. **Работает без VPN** — выделенная карточка (`featured`) с бейджем «Без VPN».
   Прямое соединение между участниками.
2. **Без ограничений по времени** — ни таймеров, ни автоотключений.
3. **Без установки приложений** — всё в браузере.
4. **Приватность по умолчанию** — шифрование, доступ по коду.
5. **Подключение в один клик** — без регистрации.
6. **Демонстрация экрана** — в высоком качестве.

Выделенная карточка имеет отдельный градиентный фон и бейдж; остальные —
обычный фон с лёгким эффектом подъёма и подсветки при наведении. Назначение
блока — донести ключевые преимущества продукта.

---

## 4. Баннер качества видео — `components/quality-banner.tsx`

Завершающая секция-«карточка» с закруглёнными углами, декоративным свечением и
сеткой. Содержит:

- бейдж «Качество видео» (с иконкой);
- крупный заголовок **«Видео в 1080p»**;
- крупный бейдж **«Бесплатно»**;
- поясняющий текст про отсутствие платных тарифов;
- список из четырёх пунктов (каждый с галочкой):
  «Чёткое видео Full HD 1080p», «Кристально чистый звук»,
  «Без водяных знаков и рекламы», «Доступно каждому участнику».

Назначение — подчеркнуть, что высокое качество доступно всем бесплатно.

---

## Адаптивность

Все блоки построены на flexbox/grid с адаптивными префиксами Tailwind
(`sm:`, `md:`, `lg:`). Дизайн mobile-first: навигация в хедере скрывается на
мобильных, кнопки в hero выстраиваются в столбец, сетки преимуществ и списка
качества меняют число колонок. Цвета и шрифты — см.
[`design-system.md`](./design-system.md).
